Coverage

What it looks for

The scanner focuses on token-level indicators and implementation hints that usually matter during JWT reviews.

Algorithms

Header trust issues

Highlights `alg:none`, unusual algorithm choices, and context mismatches where the expected verification model differs from the observed token.

JOSE headers

Key selection red flags

Surfaces `jku`, `jwk`, `kid`, `x5u`, and `x5c` so risky key-loading decisions are not buried in the token header.

Claims

Claim hygiene checks

Reviews `exp`, `iat`, `iss`, `aud`, payload size, and authorization-flavored claims that should never be trusted without signature validation.
